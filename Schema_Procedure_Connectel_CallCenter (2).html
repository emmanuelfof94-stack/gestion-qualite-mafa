🎯 PROCÉDURE CONNECTEL - CALL CENTER

Workflow complet avec appels pré-visite et post-visite

1
📊 CHARGEMENT DES DONNÉES

Action : L'administrateur charge le fichier Excel contenant les informations des entrepreneurs collectées par les agents terrain.

📝 Formulaires pré-intégrés dans l'application :
  • Formulaire pré-visite : Confirmation RDV, vérification infos, nature activité
  • Formulaire post-visite : Satisfaction, feedback, problèmes, modifications business
↓
2
⏰ DISPATCHING AUTOMATIQUE 9H00

Action : Chaque matin à 9h00 précises, le système lance automatiquement le dispatching des fiches.

👤 GESTION DYNAMIQUE DES AGENTS PAR L'ADMIN

L'administrateur peut modifier la liste des agents actifs AVANT le dispatching de 9h :

➕ AJOUTER UN AGENT :

  • Nouvel agent recruté
  • Agent de retour (fin congé, fin arrêt maladie)
  • Agent réaffecté au call center

➖ RETIRER UN AGENT (avec motif obligatoire) :

  • 🚫 Arrêt d'activité (démission, licenciement, fin contrat)
  • 🏖️ Congé (congé annuel, congé sans solde)
  • 📋 Permission (autorisation absence courte durée)
  • 🤒 Arrêt maladie prolongé
  • 📚 Formation
  • 🔄 Réaffectation (mutation vers autre poste/projet)
  • ⏸️ Suspension temporaire
  • 📝 Autre (à préciser)

📊 TRAÇABILITÉ DES MODIFICATIONS :

Chaque ajout/retrait est enregistré avec :

  • Date et heure de la modification
  • Nom de l'agent concerné
  • Type d'action (Ajout / Retrait)
  • Motif (obligatoire pour retrait)
  • Admin ayant effectué la modification
⚙️ RÉPARTITION DYNAMIQUE :

Environ 100 fiches par agent - Le nombre exact dépend du volume total disponible ET du nombre d'agents actifs

Exemple :

  • 1700 fiches ÷ 17 agents actifs = 100 fiches/agent
  • 1700 fiches ÷ 15 agents actifs (2 retirés) = 113 fiches/agent
  • 1800 fiches ÷ 18 agents actifs (1 ajouté) = 100 fiches/agent
✅ LISTE DE PRÉSENCE AUTOMATIQUE :

Génération à 9h00 : Le système crée automatiquement la liste avec :

  • Nom de l'agent
  • Heure de connexion
  • Volumétrie attribuée (fiches reçues)
  • Statut (Présent / Absent + motif / Retiré + motif)

⚙️ Mise à jour automatique à 17h30 : La même liste est actualisée directement dans le système avec l'ajout de la colonne "Nombre total traité" pour chaque agent.

↓
3
👤 CONNEXION DES AGENTS

Action : Les agents se connectent à leur compte Connectel.

🔀 DEUX SCÉNARIOS :
✅ CONNECTÉ AVANT 9H

→ Reçoit fiches automatiquement à 9h00

→ Inclus dans la liste de présence

→ Statut : PRÉSENT

⏰ CONNECTÉ APRÈS 9H

→ Reçoit fiches dès connexion

→ Répartition recalculée automatiquement

→ Heure de connexion enregistrée

→ Admin renseigne motif du retard

📋 Gestion des absences par l'administrateur :

⚠️ IMPORTANT : C'est l'administrateur qui renseigne les motifs d'absence/retard des agents dans le système.

Motifs disponibles :

  • 🤒 Maladie
  • 🏖️ Congé
  • 🚨 Urgence personnelle
  • ⏰ Retard
  • ☕ Pause
  • 📝 Autre (à préciser)

Note : Ces motifs sont renseignés par l'admin en début de journée ET en cours de journée si nécessaire.

↓
4
📞 PHONING PRÉ-VISITE

Objectifs :

  • ✅ CONFIRMER le rendez-vous
  • ✅ VÉRIFIER les informations de base
  • ✅ CONFIRMER la nature de l'activité
  • ✅ EXPLIQUER l'objet de la visite
  • ✅ OPTIMISER le planning terrain

📊 LES 3 NIVEAUX DU PHONING

NIVEAU 1️⃣ - POINTAGE PAR L'AGENT

Action : L'agent remplit le formulaire pendant/après l'appel

  • Statut de l'appel (Joignable/Injoignable/Occupé/Refus)
  • Confirmation RDV (Oui/Non)
  • Date et heure RDV
  • Observations

→ Ce que l'agent déclare avoir fait

NIVEAU 2️⃣ - ENREGISTREMENT AUTOMATIQUE

Action : L'IA Connectel enregistre automatiquement l'appel

  • 📼 Enregistrement vocal complet de la conversation
  • Stockage automatique dans le système
  • Horodatage précis (début + fin)
  • Durée de l'appel

→ Preuve objective de ce qui s'est réellement passé

NIVEAU 3️⃣ - VÉRIFICATION CONFORMITÉ

Action : L'IA Connectel analyse l'enregistrement et compare avec le script

  • ✅ Analyse automatique de l'enregistrement
  • ✅ Vérification des 5 étapes du script
  • ✅ Détection des questions obligatoires
  • ✅ Champ "Script conforme" : OUI / NON

→ Validation automatique de la qualité de l'appel

🔍 COMPARAISON NIVEAU 1 vs NIVEAU 2 :

L'IA compare automatiquement ce que l'agent a déclaré (Niveau 1) avec ce qui s'est réellement passé dans l'enregistrement (Niveau 2) pour détecter les écarts.

📝 SCRIPT PRÉ-VISITE :

1. SALUTATION

"Bonjour, je m'appelle [Nom] du projet ID30 de MAFA HOLDING. Est-ce que je parle bien avec [Nom marchand] ?"

2. OBJET

"Je vous appelle pour confirmer le passage de notre agent demain vers [heure] pour mettre à jour vos informations."

3. CONFIRMATION

"Est-ce que vous serez disponible à ce moment-là ?"

4. VÉRIFICATION ACTIVITÉ

"Je vérifie : votre commerce est bien à [adresse], dans le secteur [activité] ?"

Questions complémentaires :

  • "Votre business existe-t-il toujours ?"
  • "Avez-vous connu des changements depuis l'enrôlement ?" (Déménagement, incendie, recrutement, changement d'activité)
5. CLÔTURE

"Très bien, notre agent passera demain. La visite prendra environ 15 minutes. Bonne journée !"

💬 GESTION DES OBJECTIONS :

Objection : "Je n'ai pas le temps"

→ "Je comprends, c'est très rapide : seulement 15 minutes. On peut proposer un autre créneau ?"

Objection : "Vous allez me demander de l'argent ?"

→ "Non, absolument pas ! C'est totalement gratuit. Il s'agit uniquement de collecter vos informations."

🤖 CONTRÔLE IA CONNECTEL - CONFORMITÉ SCRIPT PRÉ-VISITE

L'IA Connectel vérifie automatiquement :

  • ✅ Les 5 étapes du script ont été respectées
  • ✅ Les questions obligatoires ont été posées
  • ✅ Le ton est professionnel et courtois
  • ✅ Les objections ont été correctement gérées
  • ✅ La durée de l'appel est appropriée (2-4 minutes)
↓
5
📞 PHONING POST-VISITE

Délai optimal : 2 à 4 heures après la visite

Objectifs :

  • ✅ MESURER la satisfaction du marchand
  • ✅ COLLECTER des informations complémentaires
  • ✅ IDENTIFIER les problèmes survenus
  • ✅ CONFIRMER que toutes les données ont été collectées

  • Satisfaction entrepreneur (Note 1-5)
  • Ponctualité agent terrain
  • Professionnalisme (Note 1-5)
  • Type visite (Nouvel/Mise à jour)
  • Corrections demandées (Oui/Non)

→ Ce que l'agent déclare avoir collecté

Action : L'IA Connectel enregistre automatiquement l'appel

  • 📼 Enregistrement vocal complet de la conversation
  • Stockage automatique dans le système
  • Horodatage précis (début + fin)
  • Durée de l'appel

→ Preuve objective de ce qui s'est réellement passé

NIVEAU 3️⃣ - VÉRIFICATION CONFORMITÉ

Action : L'IA Connectel analyse l'enregistrement et compare avec le script

  • ✅ Analyse automatique de l'enregistrement
  • ✅ Vérification questions satisfaction posées
  • ✅ Validation type visite identifié
  • ✅ Champ "Script conforme" : OUI / NON

→ Validation automatique de la qualité de l'appel

🔍 COMPARAISON NIVEAU 1 vs NIVEAU 2 :

L'IA compare automatiquement ce que l'agent a déclaré (Niveau 1) avec ce qui s'est réellement passé dans l'enregistrement (Niveau 2) pour détecter les écarts.

📝 SCRIPT POST-VISITE :

1. SALUTATION

"Bonjour, c'est [Nom] du projet ID30. Je vous rappelle suite au passage de notre agent aujourd'hui."

2. SATISFACTION

"Est-ce que tout s'est bien passé ? Êtes-vous satisfait de la visite ?"

3. VÉRIFICATION DONNÉES COLLECTÉES

"L'agent a bien pris toutes vos informations ?"

⚠️ ATTENTION - Deux cas de figure :

📌 CAS 1 : NOUVEL ENTREPRENEUR (première visite)

"Je vois que c'est votre premier enrôlement dans notre base. Permettez-moi de confirmer quelques informations :"

  • "Vous exercez bien dans le secteur [activité] ?"
  • "Votre commerce est situé à [adresse complète] ?"
  • "Depuis combien de temps exercez-vous cette activité ?"
  • "Avez-vous des employés ?" (Si oui, combien ?)

📌 CAS 2 : ENTREPRENEUR EXISTANT (mise à jour)

"Parfait. Juste pour vérifier, il n'y a pas d'informations incorrectes que nous devrions corriger ?"

Si l'entrepreneur mentionne une erreur :

  • "Quelle information est incorrecte exactement ?"
  • "Quelle est la bonne information ?"
  • → NOTER la correction dans le formulaire avec mention "CORRECTION DEMANDÉE"
4. QUESTIONS SATISFACTION DÉTAILLÉES
  • "La durée de la visite était acceptable ?"
  • "L'agent était-il ponctuel ?"
  • "Les explications étaient-elles claires ?"
  • "Avez-vous des remarques ou suggestions ?"
5. COLLECTE FEEDBACK
Critère Options
Ponctualité agent À l'heure / Retard / En avance
Professionnalisme Note de 1 à 5
Clarté explications Oui / Non / Moyen
Durée visite Acceptable / Trop longue / Trop courte
Satisfaction globale Note de 1 à 5
Type de visite Nouvel entrepreneur / Mise à jour
Corrections demandées Oui (préciser) / Non
6. CLÔTURE

"Merci beaucoup pour votre temps et votre collaboration. Bonne journée !"

🤖 CONTRÔLE IA CONNECTEL - CONFORMITÉ SCRIPT POST-VISITE

L'IA Connectel vérifie automatiquement :

  • ✅ Toutes les questions de satisfaction ont été posées
  • ✅ Les notes ont été correctement collectées
  • ✅ Le feedback est complet
  • ✅ Type de visite identifié (Nouvel entrepreneur / Mise à jour)
  • ✅ Pour nouveaux entrepreneurs : Confirmation activité effectuée
  • ✅ Vérification demande de corrections
  • ✅ Si corrections : Détails notés clairement
  • ✅ L'appel a été fait dans le délai optimal (2-4h après visite)
  • ✅ Le ton reste professionnel et empathique

6
📊 GÉNÉRATION DOUBLE RAPPORT

Action : Suite à chaque appel (pré ou post-visite), le système génère DEUX rapports distincts.

📝 RAPPORT AGENT

Formulaire renseigné manuellement par l'agent pendant l'appel

  • Statut appel
  • Confirmation RDV
  • Réponses questions
  • Observations
🤖 RAPPORT IA

Analyse automatique de l'enregistrement vocal par l'IA Connectel

  • Transcription complète
  • Détection infos clés
  • Conformité script
  • Analyse tonalité
↓

🤖 ANALYSE IA CONNECTEL - COMPARAISON DES RAPPORTS

L'intelligence artificielle Connectel effectue une comparaison automatique entre les deux rapports pour détecter les écarts.

🔍 Types d'écarts détectés :
  • Écarts d'information : Agent note "RDV confirmé" mais entrepreneur dit "Peut-être"
  • Écarts de discours : Script non respecté, étapes manquantes
  • Écarts de conformité : Ton inapproprié, objections mal gérées
  • Écarts de données : Date/heure différente entre rapport agent et conversation réelle
  • Questions manquées : Questions obligatoires non posées (existence business, changements)
  • Nouveaux entrepreneurs : Confirmation activité non effectuée pour première visite
  • Corrections ignorées : Entrepreneur demande correction mais agent ne note pas

💡 Ces écarts sont marqués pour révision superviseur et amélioration continue.

🎯 POINTS CLÉS DU SYSTÈME

  • Dispatching flexible : ~100 fiches/agent à 9h, agents en retard reçoivent fiches dès connexion
  • Double mission : Appels PRÉ-VISITE (confirmation + vérification) et POST-VISITE (satisfaction + feedback)
  • Scripts standardisés : Conformité vérifiée automatiquement par IA
  • Questions essentielles : Existence business + Changements depuis enrôlement
  • Double validation : Rapport agent + Rapport IA pour garantir qualité
  • Détection d'écarts : L'IA identifie automatiquement les incohérences
  • Liste de présence dynamique : Créée à 9h avec volumétrie attribuée, actualisée à 17h30 avec nombre total traité
  • Suivi absences : Motifs enregistrés et consolidés (journalier + mensuel)
  • Traçabilité totale : Enregistrements, rapports, et consolidation pour analyse
